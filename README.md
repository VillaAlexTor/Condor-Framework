## ВҝPor quГ© existe este proyecto?

Durante el desarrollo de auditorГӯas de seguridad reales en entornos acadГ©micos y profesionales, se identificГі un problema recurrente: **el reconocimiento pasivo es tedioso, fragmentado y difГӯcil de documentar**.

Las herramientas actuales como Maltego, Shodan o FOCA son poderosas pero aisladas. El analista termina copiando datos entre ventanas, construyendo informes a mano y sin trazabilidad del proceso. CГіndor Framework resuelve esto al unificar la cadena completa en un ecosistema cohesionado, open source y extensible.

**Casos de uso:**
- AuditorГӯas de seguridad en organizaciones (pre-engagement)
- Ejercicios acadГ©micos de ethical hacking
- Bug bounty (reconocimiento inicial)
- Monitoreo de exposiciГіn de activos propios

---

## MГіdulos

### `condor-cli` вҖ” OSINT Automator CLI

Motor de recolecciГіn pasiva de inteligencia. Script Python con interfaz de lГӯnea de comandos que consulta mГәltiples fuentes pГәblicas y APIs sin generar trГЎfico directo al objetivo.

**Fuentes consultadas:**
- DNS pasivo (registros A, MX, NS, TXT, CNAME)
- WHOIS histГіrico y actual
- Censys.io (puertos, servicios, certificados TLS)
- Shodan (banners, tecnologГӯas expuestas)
- Hunter.io (emails corporativos filtrados)
- FOCA-like (metadatos de documentos pГәblicos vГӯa Google Dorks)
- Have I Been Pwned API (brechas asociadas al dominio)
- Wayback Machine / Archive.org (URLs histГіricas)
- Certificate Transparency Logs (subdominios via crt.sh)

**Output:** JSON estructurado, listo para ser consumido por el dashboard o exportado directamente.

```bash
# Uso bГЎsico
python condor.py --target ejemplo.bo --output recon_ejemplo.json

# Con mГіdulos especГӯficos
python condor.py --target ejemplo.bo --modules dns,whois,censys,crt

# Output HTML standalone
python condor.py --target ejemplo.bo --format html --output reporte.html

# Verbose + guardar logs
python condor.py --target ejemplo.bo --verbose --log condor.log
```

**TecnologГӯas:** Python 3.11+, `argparse`, `dnspython`, `python-whois`, `requests`, `rich` (CLI styling), `jinja2` (HTML output)

---

###  `condor-dashboard` вҖ” Passive Recon Dashboard

Interfaz web interactiva que consume el JSON generado por `condor-cli` y presenta los resultados de forma visual y navegable.

**CaracterГӯsticas:**
- Carga de archivos JSON desde el CLI o ingesta directa vГӯa API local
- Mapa de red interactivo (nodos: dominio вҶ’ subdominios вҶ’ IPs вҶ’ puertos)
- Timeline de cambios histГіricos (Wayback + DNS pasivo)
- Tabla de puertos y servicios con filtros
- Heatmap de exposiciГіn por categorГӯa de riesgo
- ExportaciГіn de vistas seleccionadas hacia `condor-report`

**TecnologГӯas:** React 18, D3.js (grafos de red), Recharts (charts), Tailwind CSS, Vite

---

###  `condor-report` вҖ” Vulnerability Ficha Generator

Generador de informes de auditorГӯa. Toma los datos del dashboard y produce fichas de vulnerabilidad formateadas con scoring CVSS 3.1 automГЎtico y recomendaciones.

**CaracterГӯsticas:**
- Fichas individuales por hallazgo con campos: descripciГіn, evidencia, impacto, vector CVSS 3.1
- Calculadora CVSS 3.1 integrada (AV, AC, PR, UI, S, C, I, A)
- ClasificaciГіn automГЎtica: CrГӯtico / Alto / Medio / Bajo / Informativo
- Recomendaciones predefinidas por categorГӯa de vulnerabilidad
- Export a PDF, DOCX y HTML
- Portada, Гӯndice, resumen ejecutivo y fichas tГ©cnicas generados automГЎticamente

**TecnologГӯas:** Node.js / Express, Puppeteer (PDF), `docx` library, Vue.js 3 (frontend del generador)

```bash

```

### OpciГіn B вҖ” Manual por mГіdulo

```bash

cd condor-framework

# --- condor-cli ---

# --- condor-dashboard ---
cd ../condor-dashboard
npm install
npm run dev

# --- condor-report ---
cd ../condor-report/backend
npm install
npm run start

cd ../frontend
npm install
npm run dev
```
